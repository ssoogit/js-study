# 2026-05-22 | 객체 생성 방법(생성자 함수 or 리터럴)

## 객체 생성 방법
생성자 함수 사용 or 리터럴 사용

객체는 중괄호로 나타난다.

## 생성자 함수
Object() 라는 생성자 함수를 사용. `Object()`

객체 생성자 함수 object는 new 라는 키워드와 함께 작성해야 함. `new Object()`

## 리터럴
중괄호를 사용하여 객체를 생성. `{}`

## 객체 프로퍼티(객체 속성)
객체의 중괄호 내부에 들어갈 데이터

키와 밸류 쌍으로 이루어짐. `key: value`
* value 값에는 undefined, 함수 등 어떠한 값이든 들어갈 수 있다.


##  객체 프로퍼티 사용 방법
* 점 표기법 `car.name`
* 괄호 표기법 `car['name']`
  * 문자열임을 나타내는 `'' or ""`를 사용
 
일반적으로 객체 프로퍼티의 값을 꺼내 사용할 때는 **점 표기법**을 많이 사용하지만,

객체의 키 값들이 고정적이지 않고 특정 함수의 매개변수에 따라 키 값을 계속해서 변경해야하는 경우에는 **괄호 표기법**을 활용한다.

-> 괄호 표기법: 객체의 키 값을 특정 함수의 매개변수가 결정하는 상황 or 키 값이 계속 변화하는 상황에서 자주 사용

## 객체 프로퍼티 추가 및 수정, 삭제 방법
* 사용할때와 같이 점 표기법 or 괄호 표기법 사용 (예제 코드 참고)
* 객체는 const로 선언하더라도, 프로퍼티의 값은 수정할 수 있다.
  * Why? -> 객체 프로퍼티 값을 수정하는 것과 cat이라는 객체 자체를 수정하는 것은 다른 것이기 때문! (객체 프로퍼티의 값을 수정하는 것은 객체의 고유 ID값을 변경하는 것이 아니기 때문이다.)

## 객체의 프로퍼티 값이 함수일 경우
 이를 메서드(method)라고 부른다. ( 메서드 = 값이 함수인 프로퍼티 )
 
 객체 내부의 프로퍼티들에 접근할 수 있는 기능이 있다. 

 ### 메서드를 생성할 때는 화살표 함수보다는 function 키워드를 사용해서 생성하는 것이 좋다.
 -> 화살표 함수 내부에 this라는 키워드는 해당 메서드를 호출한 객체를 가리키지 못하고, 가장 바깥에 있는 글로벌 객체인 윈도우 객체를 가리키기 때문.
