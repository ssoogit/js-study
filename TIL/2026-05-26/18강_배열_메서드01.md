## 배열 내장 함수
내장함수, 메서드( push 와 unshift 같은)

자주 사용되는 내장함수들

일단 포문을 사용하여, 배열의 모든 요소에 접근하는 코드 작성 예시를 보여줌.

배열 내장 함수를 사용하여 작성하면 더 쉽게 접근 가능

### [ 배열을 순회해야할 때 사용하는 내장함수 ]

#### 1. forEach 배열 내장 함수 `arr.forEach();`
* 이 메서드에는 매개변수로 함수를 전달할 수 있다.(함수를 매개변수로 전달하는 개념: 콜백(callback)함수)
* forEach 괄호 안에 콜백 함수를 화살표 함수로 작성.
* 이 콜백함수는 3가지의 매개 변수를 전달할 수 있다.
  * CurrentValue: 처리할 현재 요소(배열의 각 요소를 가리켜용) 
  * index: 실제 배열 요소의 순서를 나타내는 인덱스 전달 (선택적)
  * array: 실제 배열을 배열 요소의 수만큼 출력
* 배열에서 for문을 대체하여 사용
* 반환값이 없다. undefined를 반환한다. 새로운 배열이 필요하다면 map 사용
* 중단 할 수 없다.
* 원본 배열 수정 가능하다.

#### 2. map
* 전달한 콜백 함수를 호출한 결과를 모아서 새로운 배열로 반환해주는 내장 함수
* 배열의 모든 요소에 연산을 적용하고, 새로운 배열로 돌려받을 수 있다.
* 3가지의 매개변수를 전달할 수 있다.( forEach와 동일)

### [ 배열에서 특정 요소를 찾는 함수 ]

#### 1. at
* 배열의 가장 마지막 요소에 쉽게 접근하게 해주는 메소드.
* 음수 인덱스를 지원한다는 특징이 있다(일반적인 배열 인덱스는 음수 인덱스 입력 시 undefined반환)

#### 2. includes `includes(확인요소, 작성된 숫자 이후(포함)에 존재하는지)`
* 매개변수로 받은 요소를 배열이 포함하고 있는지 판별하여 분리한 값으로 반환.
* 찾아봐야알겟지만, 아마 이것 또한 배열 요소의 값이 객체 형태이거나 배열의 형태일 때는 못 찾아용


#### 3. indexOf `indexOf(확인요소, 작성된 숫자 이후(포함)에 존재하는지)`
* 특정값을 지닌 요소가 몇 번째에 위치(0부터 시작)하는 요소인지 찾아준다.
* 존재하지 않는 값을 찾는 경우 -1을 반환.
* 배열 요소의 값이 객체 형태거나 배열의 형태일 때의 인덱스는 못 찾음.
 

#### 4. findIndex - 콜백함수 작성
* 내부 요소들이 모두 객체 형태 일때(설명 더 찾아봐야할듯), 원하는 요소가 배열의 몇번째 요소인지 찾는데 사용
* 조건을 가장 먼저 만족하는 배열 요소의 인덱스를 반환
 

#### 5. find
* 찾아낸 값 그 자체를 반환하는 함수

### [ 배열에서 특정 요소들을 추출할 수 있는 메서드 ]

#### 1. filter
* 배열에서 특정 조건을 만족하는 값들만 따로 추출해서 새로운 배열을 생성할 수 있다.
 

#### 2. slice `.slice(Begin, End)`
* 배열에서 원하는 부분만 잘라서 가져와 새로운 배열을 생성할 수 있다.
* 전달받은 매개변수(Begin, End), Begin부터 End보다 1 작은 인덱스 요소까지 배열을 복사해서 새로운 배열 객체로 반환.
